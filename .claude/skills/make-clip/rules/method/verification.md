# 프레임 검증 게이트 — 렌더 전 필수 (이 단계가 품질을 만든다)

video-production의 교훈: **검증 단계가 영상을 좋게 만든다.** 드래프트/스틸을 뽑아 **실제로 눈으로 본다.** 생략하면 "PPT 범벅·잘린 음성·깨진 레이아웃"이 그대로 양산된다.

## STILL QA (싸다, 빠르다)
```bash
node factory/build.mjs <id> --no-video
node factory/still.mjs <id> 245,1225,3185,8575     # 역할이 서로 다른 씬들의 중간 프레임
```
→ `factory/_verify/still_<id>_<frame>.png`를 **Read 툴로 직접 본다.**

## 체크리스트 (씬마다)
- [ ] 텍스트 가독성 — 작지 않은가, 대비 충분한가.
- [ ] 오버플로/잘림 없음 — 카드 밖으로 안 넘침, 줄바꿈 정상.
- [ ] 역할 오탐 없음 — 의도한 레이아웃(표지/용어/프로세스/숙제/코드/정리/진술/리스트)이 맞게 떴는가.
- [ ] 브랜드 색 — accent/bg가 의도대로.
- [ ] **음성 길이 ≈ 비주얼 길이** — 사이드카 durationMs가 내레이션 분량과 맞는가(잘림 없음). [[fal-tts-truncation-bug]]
- [ ] 자막 — 문장 단위 한 줄씩, 6줄 덩어리 아님.

## 고칠 때 — 바뀐 씬만
문제 있는 씬만 project.json 수정 → 그 씬 음성만 재생성(sig 캐시로 나머진 그대로) → 다시 still 확인. **전체 파이프라인 재실행 금지**(느리고 낭비).

## 통과 후에만 렌더
```bash
node factory/render.mjs <id>          # GPU
node factory/validate-course.mjs <id> # 자문용(슬라이드 지배형 image 경고는 오탐 가능)
```

## 양산 규칙
**한 편을 still 검증으로 레퍼런스 확정한 뒤에만 확장.** "설계만 하고 바로 양산"이 이번 시즌 품질 저하·비용 낭비의 근본 원인이었다(고정 틀 + 검증 생략 + 음성 미확인).
