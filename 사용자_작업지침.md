# 슬라이드 제작 작업 지침서 v2 (사용자용)

> Gemini × Antigravity 교안 120 슬라이드 제작 프로젝트
> **Claude Design + Claude Code 하이브리드 워크플로우**

---

## 0. 큰 그림

```
[Claude Design]                    [Claude Code]
디자인 시스템 구축 (1회)    →     핸드오프 번들 받기
샘플 슬라이드 3장 만들기   →     나머지 117장 일괄 제작
                                  HTML → PPTX 변환
```

**핵심 원칙 4가지**:
1. **Claude Design을 디자인 시스템 박제기로 사용** — 톤·컬러 1회 정의
2. **샘플 3장만 Claude Design에서** — 사용량 절약 + 핸드오프 기준
3. **나머지는 Claude Code 일괄** — 모듈 단위로 끊어서
4. **핸드오프 번들이 단일 진실 원천** — 모든 디자인 결정의 출처

## 1. 두 가지 경로 (본인 상황 확인)

### 경로 A: Claude Design 접근 가능 ⭐ 권장
- claude.ai/design 접속 가능
- Pro/Max/Team/Enterprise 구독자
- → **이 지침서의 Phase 1~6 그대로 진행**

### 경로 B: Claude Design 접근 불가
- 아직 롤아웃 전이거나 기능 미노출
- claude.ai/design 접속 시 안 뜸
- → **Phase 2~4 건너뛰고 부록 A의 대체 경로 사용**

**확인 방법**: claude.ai 로그인 후 https://claude.ai/design 접속 → 캔버스 화면이 뜨면 경로 A, 안 뜨면 경로 B

## Phase 2. Claude Design 디자인 시스템 구축 (2시간) ⭐

> **이 단계가 전체 품질의 80%를 결정합니다. 절대 건너뛰지 마세요.**

### 2-1. claude.ai/design 접속
- 좌측 하단 조직 이름 클릭 → "코드코리아" 조직 선택 또는 신규 생성
- 자동으로 온보딩 플로우 진입

### 2-2. 브랜드 자료 업로드
다음 자료를 모아서 업로드:
- 이전에 만든 이력서 DOCX/PDF (보태니컬 톤)
- Behance 참고 이미지 스크린샷
- 만족스러웠던 기존 제안서·발표자료
- 코드코리아 로고/색상 가이드 (있다면)

**팁**: 큰 폴더 통째 X → 핵심 파일 5~10개만 선별 업로드 (사용량 절약)

### 2-3. 디자인 시스템 추출 검증
업로드 후 Claude가 자동으로 컬러·타이포·컴포넌트 패턴을 추출.

검증용 테스트 프롬프트:
```
"코드코리아의 표준 슬라이드 1장을 만들어줘.
제목·본문·인포그래픽 카드 3개·푸터가 있는 16:9 슬라이드."
```

결과가 본인 브랜드답게 나오는지 확인 → OK면 **"Published" 토글 ON**

## Phase 3. 샘플 슬라이드 3장 제작 (2시간)

### 3-1. 새 프로젝트 생성
"Slide deck" 선택 → 다음 프롬프트 (curriculum_plan_v2.md 첨부):

```
Gemini × Antigravity 12시간 교육 슬라이드 덱.
첨부한 curriculum_plan_v2.md를 참고해서 다음 3장만 먼저 만들어줘:

1) 슬라이드 1번 (표지)
   - "Gemini × Antigravity AI Agent IDE 실전 활용 과정"
   - 부제: "프롬프트 엔지니어링부터 컨텍스트·하네스 엔지니어링까지"
   - 12시간 / 120 슬라이드 표시

2) 슬라이드 48번 (워크플로우 다이어그램)
   - "자료조사 자동화 8단계 워크플로우"
   - 8개 단계를 시각적으로 연결한 다이어그램

3) 슬라이드 82번 (HTML-First 5대 이유)
   - "왜 HTML로 먼저 만들고 변환하는가"
   - 토큰효율·시각검증·반복비용·일관성·자유도 5개 카드

조건:
- 16:9, 한국어
- 코드코리아 디자인 시스템 적용
- 슬라이드 우측 하단에 번호 + "Gemini × Antigravity" 표기
- 각 슬라이드 헤더에 모듈 표시 ("M1", "M5" 등)
```

### 3-2. 변형 요청 + 비교
```
표지 슬라이드 3가지 다른 방향으로 만들어줘:
1) 미니멀 화이트 + 강한 타이포
2) 보태니컬 그린 + 크림 (메인 브랜드 톤)
3) 다크 + 액센트 그린
```

### 3-3. 인라인 코멘트로 미세 조정
- 캔버스에서 특정 요소 클릭 → 코멘트 ("이 간격 8px로 줄여")
- 구조 변경은 좌측 채팅 ("3번 슬라이드 카드 배치를 2x3 그리드로")

### 3-4. 최종 컨펌
3장이 마음에 들면 다음 단계로

## Phase 4. Claude Code로 핸드오프 (1시간)

### 4-1. Export → Send to Claude Code
- 우측 상단 Export 버튼 클릭
- "Send to Claude Code" 선택 (터미널 사용 시)
- 또는 "Send to Claude Code Web" (웹 사용 시)
- 핸드오프 번들 다운로드

### 4-2. 번들 압축 해제
```bash
unzip claude-design-bundle.zip -d ./handoff/
ls handoff/
# tokens.css, components/, samples/, meta.json 등 확인
```

### 4-3. Claude Code 첫 명령
```
@CLAUDE.md @docs/curriculum_plan_v2.md @handoff/

handoff/ 폴더에 Claude Design에서 받은 번들이 있어.
다음 작업해줘:

1. handoff/tokens.css를 styles/tokens.css로 복사
2. handoff/samples/ 의 3장 분석해서 패턴 추출
3. 그 패턴 기반으로 slide-template.html, components.html 생성
4. design-system.md 작성 (handoff/meta.json + 추출 패턴 종합)
5. 50번 슬라이드 1장 제작해서 샘플과 톤 일치하는지 검증
6. 스크린샷 보여주고 멈춰

샘플과 일관성 확인 후 모듈 일괄 진행한다.
```

## Phase 6. PPTX 변환 + 검수 (2시간)

### 두 가지 옵션

**옵션 1: Claude Design에서 직접 export**
- 모든 슬라이드를 Claude Design에서 만든 경우 (사용량 풍부할 때)
- Export → PPTX 선택 → 즉시 다운로드
- 장점: 깔끔, 단점: 사용량·시간 많이 소모

**옵션 2: Claude Code에서 HTML → PPTX 변환** ⭐ 권장
```
모든 슬라이드 HTML 완성 완료.
.claude/skills/pptx/SKILL.md 읽고 다음 작업해줘:

1. slides/ 아래 모든 HTML을 1920x1080 PNG로 Puppeteer 캡처
2. python-pptx로 16:9 PPTX 생성
3. 슬라이드 번호 순서대로 이미지 풀사이즈 삽입
4. 발표자 노트에 curriculum_plan_v2.md의 "핵심" 자동 입력
5. output/curriculum.pptx 저장

완료 후 LibreOffice로 첫·중간·마지막 슬라이드 스크린샷 보여줘.
```

### A-1. claude.ai 일반 채팅에서 샘플 만들기
```
[새 채팅에서]
첨부한 curriculum_plan_v2.md 기반으로 다음 슬라이드 3장 HTML로 만들어줘:
1) 표지
2) 워크플로우 다이어그램 (48번)
3) HTML-First 5대 이유 (82번)

코드코리아 톤: 보태니컬 그린(#2D5F3F)·크림(#F5F0E8) 베이스, Pretendard 폰트.
16:9, 1280x720, 단일 HTML 파일 3개.
```

### A-2. 만족스러운 결과를 reference로 박제
```bash
mkdir -p docs/reference
# 가장 좋은 3개 HTML 파일 docs/reference/ 에 복사
```

### A-3. Claude Code 첫 명령 (handoff/ 대신 reference/ 사용)
```
@CLAUDE.md @docs/curriculum_plan_v2.md @docs/reference/

reference/ 의 3장이 우리 디자인 표준이야.
1. 3장 분석해서 design-system.md, styles/tokens.css 추출
2. slide-template.html, components.html 생성
3. 50번 슬라이드 샘플 제작해서 reference 톤 일치 확인
4. 스크린샷 보여주고 멈춰
```

이후 Phase 5~6은 동일 진행.
